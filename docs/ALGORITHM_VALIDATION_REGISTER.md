# MELE 演算法驗證登錄表

最後更新：2026-04-30

## 1. 原則

命理系統存在流派差異。MELE 的文件必須清楚區分：

- 已由公式確定的部分。
- 已與第三方工具交叉驗證的部分。
- 採用特定流派或常見版本的部分。
- 尚未充分實證的灰色地帶。

不得把「目前可用」寫成「絕對官方正確」。

## 2. 工具級驗證狀態

| 工具 | 目前可信度 | 灰色地帶 | 上線前補強 |
|---|---:|---|---|
| 生命靈數 | 高 | 文本個人化不足 | 補 1-9、11、22、33 的深度文本與命理師審稿 |
| 馬雅 | 中高 | Guide / Oracle 部分公式樣本不足 | 增加 tones 5、10 與 family offset 的實證案例 |

## 3. 固定案例格式

每個確定性工具的驗證案例應保留：

```text

時區：

第三方工具 B：
MELE 結果：
差異：

驗證日期：

```

## 4. 不一致處理

若不同來源結果不一致：

1. 不直接改程式迎合單一網站。
2. 確認時區、曆法、真太陽時、夏令時間、流派設定。
3. 記錄每個來源的結果。
4. 產品頁用「採用版本」說明，而非宣稱其他版本錯誤。

## 5. 塔羅與盧恩驗證

隨機工具不驗證「抽到哪張一定正確」，而是驗證：

- seed 相同時可重現。
- 無重複抽牌或重複符文。
- 正逆位機率符合設定。
- 每張牌/符文都有中文名、英文名、關鍵字、正位、逆位與簡短建議。
- AR 呈現與抽到的牌/符文一致。

## 6. 上線前最低門檻

- 每個確定性工具至少 10 組案例。
- 每個隨機工具完整資料庫檢查。
- 灰色地帶在產品或文件中明確揭露。
- 命理老師或熟悉該系統的人完成文本審閱。
